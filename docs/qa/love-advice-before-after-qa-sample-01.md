# Love Advice Pipeline Before/After QA — Sample 01

## 1. 테스트 조건

| 항목 | 값 |
|---|---|
| 샘플 | 1990-01-01 12:00 M / E2E love dummy chart |
| 모델 | gpt-4o-mini |
| 비교 방식 | 동일 입력에서 새 레이어 제거 전/후 비교 |
| 대상 섹션 | 1. 기본 성향 / 2. 끌림-적합 / 6. 지금 당장 해야 할 준비 |
| 산출물 | `docs/qa/love-advice-before-after-sample-01.json` |

## 2. 정량 비교

| 지표 | Before | After | 판정 |
|---|---:|---:|---|
| 글자 수 | 1202 | 1112 | After가 더 압축됨 |
| 섹션 수 | 3 | 3 | 동일 |
| 내부 자미두수 용어 노출 | 25 | 0 | 개선됨 |
| 내부 rule/source 노출 | 0 | 0 | 유지 |
| 금지 프레임 노출 | 0 | 0 | 유지 |
| 실행 행동 신호 | 5 | 7 | 개선됨 |

## 3. 기준별 판정

| 항목 | 판정 | 근거 |
|---|---|---|
| 해석 가이드 준수 | 통과 | 섹션 구조와 해석 축이 기존 love guide 범위 안에 머문다. |
| 문제점 선명도 | 부분 개선됨 | After는 `조건 기준과 감정적 욕구의 불일치`, `감정 확인 루프`가 더 직접적으로 보인다. |
| 조언 실용성 | 개선됨 | After는 `기준을 언어화`, `양보 가능/불가능 정리`, `대화의 질`처럼 실행 단위가 늘었다. |
| 과잉 조언 방지 | 통과 | 섹션 1, 2는 해석 중심이고 섹션 6에만 실행 조언이 집중된다. |
| 섹션별 rule 제한 | 통과 | context에 전체 최대 6개, 섹션 1~5 최대 2개, 섹션 6 최대 3개 제한이 적용된다. |
| 금지 프레임 | 통과 | 성별 본질론, 정복/소유/밀당/조종/서열화 표현 없음. |
| 자미두수 용어 노출 | 통과 | 품질 비교 중 발견된 용어 노출 문제를 보정했고 After 샘플에서 0건 확인. |
| rule 노출 방지 | 통과 | `rule_id`, `source_book`, `source_excerpt`, `C-` 노출 없음. |

## 4. 발견 이슈와 조치

| 이슈 | 조치 |
|---|---|
| 최초 After 샘플에서 별/궁 이름이 일부 노출됨 | `LOVE_ADVICE_PIPELINE_GUARDRAILS`에 `최종 출력 용어 변환 규칙` 추가 |
| love 테마가 기존 `commonRules`의 전문용어 금지 규칙을 직접 상속하지 않음 | love 전용 system prompt에 별도 용어 변환 규칙 추가 |
| E2E가 love system prompt의 용어 변환 규칙을 검증하지 않음 | `backend-report-prompt.spec.ts`에 positive assertion 추가 |

## 5. 결론

After가 Before보다 연애 조언 데이터의 목적에 더 맞는다.

```text
Before: 해석은 가능하지만 별/궁 이름이 많이 노출되고, 실행 조언이 일반론에 가까움.
After: 내부 용어 노출이 0건으로 줄고, 섹션 6의 행동 조언이 더 구체화됨.
```

현재 기준에서는 rule 수를 줄이거나 늘릴 필요가 없다.

```text
권장값 유지:
- 섹션 1~5: 최대 2개
- 섹션 6: 최대 3개
- 전체: 최대 6개
```
