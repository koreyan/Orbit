# Obit Hermes Working Rules

이 파일은 Hermes 전용 운영 규칙입니다.
공통 구현 규칙은 반드시 `AGENTS.md`와 함께 읽습니다.

## Startup Checklist

세션 시작 시 아래를 먼저 확인합니다.

1. `git branch --show-current`
2. `git status --short`
3. `AGENTS.md`
4. `PRD.md`
5. `.agents/rules/code-style-guide.md`

## Session Conventions

- 세션 제목 규칙: `obit/<task>`
- 작업 계획 문서 경로: `.hermes/plans/YYYY-MM-DD_HHMMSS-<slug>.md`
- 작업 종료 보고 형식:
  1. 요약 3줄
  2. 변경 파일
  3. 검증 결과
  4. 다음 작업

## Execution Rules

- 추측으로 파일을 만들지 말고 관련 구현 파일을 먼저 읽습니다.
- 수정 전후로 영향 범위를 짧게 정리합니다.
- 여러 파일을 바꿀 때는 먼저 작은 단계로 쪼갭니다.
- 완료 보고 전에는 검증 명령을 실제로 실행합니다.
- 검증 실패 시 실패 로그를 숨기지 않고 바로 보고합니다.

## Obit Migration Defaults

- Antigravity에서 하던 스타일 규칙은 저장소 문서로 승격해 유지합니다.
- Hermes memory에는 장기 사실만 저장합니다.
- 작업 계획과 세션 단위 맥락은 `.hermes/plans/`와 session history로 분리합니다.
- 승인 없는 외부 서비스 변경, 토큰 변경, 배포 설정 변경은 하지 않습니다.

## Context Storage Rules

- 저장소 공통 규칙: `AGENTS.md`
- 제품 요구사항 기준: `PRD.md`
- Hermes 전용 운영 규칙: `.hermes.md`
- 장기 사용자/환경 선호: Hermes memory
- 지난 작업 회수: `session_search`
- 실행 계획과 다음 액션: `.hermes/plans/*.md`
- 작업 로그나 임시 진행상황은 memory에 저장하지 않습니다.

## Verification Order

가능하면 아래 순서로 확인합니다.

1. `npm run lint`
2. `npm run typecheck`
3. `npm run build`
4. 필요 시 Playwright 또는 수동 검증

아직 없는 스크립트는 먼저 추가한 뒤 실행합니다.
